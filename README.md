# 社媒内容项目交付助手 Skill

> 把零散的甲方社媒内容 Brief、沟通记录或项目现状，整理成项目负责人可直接推进的交付方案。

## 这个 Skill 解决什么

接到客户的小红书 / 抖音 / 公众号内容需求时，PM 常遇到：

- Brief 模糊，不知道项目能不能接、能不能启动
- 信息不完整，但不清楚要追问什么
- 需要带创作者、写创作手册、设计培训，但不知道从哪起手
- 项目执行混乱，进度 / 质量 / 返工失控

这个 Skill 把上述场景转成可执行的：**项目阶段判断 + 信息完整度检查 + 风险识别 + 角色分工 + 下一步动作 + 可直接复用的交付物**。

## 适用场景

按三类典型入口设计了路由：

| 入口 | 典型表达 |
|---|---|
| **商务型PM** | "这个项目能不能接 / 还缺什么 / 怎么继续沟通" |
| **内容负责人 / 内容导师** | "要带创作者 / 写创作手册 / 设计培训 / 定质检" |
| **交付PM** | "进度乱 / 返工多 / 怎么验收 / 怎么复盘" |

可生成 14 种交付物，包括：项目需求确认单、客户追问清单、项目启动检查报告、内部启动会对齐清单、角色分工表、试稿方向建议、创作手册初稿、培训安排、首轮测试任务、任务分配说明、内容质检清单、周度复盘模板、结案复盘要点、知识资产沉淀建议。

## 安装

```bash
git clone https://github.com/iceewang-gif/socialmedia-content-delivery-pm.git
mkdir -p ~/.claude/skills/
cp -r socialmedia-content-delivery-pm ~/.claude/skills/
```

重启 Claude Code，Skill 会被自动识别。


## 使用方式

### 自动触发（推荐）

用自然语言描述项目情况，Claude Code 会自动调用：

- "我刚接到一个小红书 Brief，帮我看看能不能接，还缺什么"
- "这个项目要怎么分工？怎么排期？"
- "试稿反馈来了，下一步该怎么调"
- "帮我写一份创作手册 / 质检清单 / 周度复盘模板"
- "项目返工很多，帮我看看哪里出问题了"

### 显式触发

在 Claude Code 中输入 `/socialmedia-content-delivery-pm`，再补充项目情况。

### 不会触发的场景

- 让 Claude 写单条笔记 / 标题 / 文案
- 单纯问社媒知识

## References 设计

`references/` 在 SKILL.md 中被分阶段引用，构成 Skill 的方法论支撑。下面公开的是每份 reference 在系统中的角色、触发场景与结构骨架，不含原文内容。

### 1. 项目需求初步确认单模板

- **用途**：项目初期客户需求收集 + Brief 完整度判断
- **触发场景**：Workflow 步骤 3 信息完整度检查；用户问"能不能启动 / 还缺什么 / 能不能试稿"
- **结构骨架**：项目基础信息 / 内容交付初步意向 / 产品 Brief / 目标用户与使用场景 / 核心卖点 / 参考案例 / 素材包 / 合规要求 / 交付数量与时间节点 / 审稿方与修改轮次 / 内部判断区
- **关键概念**：内部判断区 4 字段——需求清晰度判断 / 建议推进阶段 / 主要风险初判 / 是否需要创作导师提前介入

### 2. 社媒内容交付项目标准化运营手册

- **用途**:项目交付的全流程标准化指南（最核心方法论文档）
- **触发场景**：Workflow 步骤 4 阶段判断；输出阶段相关交付物（启动、试稿、验收、复盘）
- **结构骨架**：8 个项目阶段划分 / 启动条件 / 角色职责 / 试稿流程 / 内部启动会议程 / 创作SOP开发 / 培训与考核 / 内容生产与审核 / 验收发布 / 结案复盘与知识沉淀
- **关键概念**：8 阶段（初步接洽→试稿→签约→团队组建→SOP开发→生产→验收→复盘）/ 6 角色（商务、项目策划、内容导师、创作者、内容审核、人事）

### 3. 产品-用户调研分析

- **用途**：把产品/服务转成内容选题与客户追问问题的分析框架
- **触发场景**：输出客户追问清单 / 目标用户、卖点、对标案例分析
- **结构骨架**：用户画像拆解 / 用户痛点提炼 / 偏好表达形式 / 价值点优先级排序 / 对标案例评估
- **关键概念**：用户是谁 / 痛点是什么 / 偏好什么表达 / 哪些价值点值得放大

### 4. 内容运营阅读指南

- **用途**：判断内容策略方向、用户精准度、内容结构
- **触发场景**：输出内容策略方向 / 精准度/钩子素材
- **结构骨架**：内容策略定位 / 用户精准度判断 / 内容结构拆解 / 泛素材与钩子素材搭配 / AI 与人工分工边界 / 内容策略风险提示
- **关键概念**：选题/信息量/内容形式 的瓶颈定位 / 封面、标题、开头、结尾的影响权重

### 5. 笔记生产指南

- **用途**：素人笔记的标题/正文/封面/关键词/标签规范
- **触发场景**：输出创作手册中的执行标准 / 创作者培训重点 / 内容质检清单
- **结构骨架**：标题规范 / 正文结构模板 / 封面真实感与压字规范 / 关键词与标签规则 / 新手常见错误 / 交稿前质检清单
- **关键概念**：标题字数与口语化要求 / 禁忌词 / 封面真实感

### 6. Prompt 写作指南

- **用途**：项目AI工作流的 Prompt 编写规范，配套创作者使用
- **触发场景**：项目含AI环节、为创作者编写生成标题/正文/封面文案/选题的Prompt
- **结构骨架**：角色定义 / 任务目标 / 输入信息 / 输出格式 / 限制条件 / 风格要求 / 质检标准
- **关键概念**：把模糊需求转成可执行 Prompt 指令

> 此外还有一份 `参考使用说明.md`，是 references 内部索引（按场景定位该读哪份）。
